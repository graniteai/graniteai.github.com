Getting started.

1. Load a training dataset

In this tutorial, we will train a machine learning model to predict whether a passenger on the Titanic will survive the crash using the Kaggle dataset.

2. Select your target feature

Select "Survived" as the feature you want to predict, using the dropdown menu or by clicking on the table. This is a 1 if the passenger survived, and 0 if not. Set the feature type as categoric, as we want to classify passengers as having survived or not.

3. Choose how to score models

We will rank models based on their accuracy. The advanced options allow you to choose the method for scoring. Since we are interested in building a predictive model, we want to see how well our model will perform on new data. By default, we use 5-fold cross validation, but for larger datasets using Train-Test splits is sufficient. In addition, we holdout some of the data as a final check on model performance.

Analysing the data.

4. Looking at trends

Click on a feature in the table to see how it influences the proportion of passengers that survived. For numeric features, data is binned into a histogram, then the mean value or proportion is plotted. For fare it looks like passengers that paid more were more likely to survive.

5. Select the features to use

Next, we need to choose the features our models will use to predict survival. Most machine learning models have automatic feature selection so we should keep in as many as possible, but in some cases we can't use features, for example, it may be illegal to use sex or ethnicity to price insurance premiums. In this case, we will remove "PassengerID" as it is just a label.

We will also change "Pclass" to be categoric, as it is a category of passenger tickets, and "Name" to text. Our models make new features from text variables by counting the occurrence of each word, so we can automatically extract the passengers title and surname.

6. Get the models

Net, get models using the features we have selected and set a maximum training time for each model. Since this is a small dataset we will use 2 minutes, but all models can be trained for longer later on.

Training and evaluating models.

7. Train the models

To train the models, click the train button in the tables or the train all button below. The training screen shows the progress of training over time and is used to extend model train time.

Granite AI optimises the model parameters, such as decision tree depth, using an exploration vs exploitation scheme. Each dot in the training plot represents the results of a new parameter run.

8. Model Summary

The model summary shows how well the model is able to capture trends in the data. Predictions are sorted by probability and binned, then the proportions of the class in each bin is compared to the actual data.

9. Feature dependence

Using the model, we are able to remove the effects of correlated variables. By clicking on each feature, we see how well our model is able to capture the trend.

The partial plot shows how the prediction varies with our feature when all other features are at the mean value. In this case, a passenger with a higher fare has a lower chance of being in category 0, so more likely to survive.

10. ROC curve

The receiver operating characteristic curve shows the true positive rate against the false positive rate, for varying probability thresholds. The area under the curve (AUC) is a measure of precision, with a perfect score having an area of 1.

11. Confusion matrix

The confusion matrix shows the count of predicted classes against actual classes. It is useful for seeing which classes errors are occurring in.

Making predictions.

12. Interactive mode

The interactive mode lets you input new features and see the model predictions. Use this to get an understanding of the features driving the model and get real time predictions.

13. Saving predictions

The prediction tab allows you to open a new dataset and make predictions on it. Saving the results will create a new file with the original dataset and predictions as a new column. All Granite AI projects can be saved and shared, so anyone can use your trained models.
